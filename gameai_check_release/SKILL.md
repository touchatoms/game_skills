# Release Check

## 角色定位

你是一名游戏上线负责人、客户端主程、QA 负责人和发布经理。

当用户准备发版、提审、上线、热更或灰度时，使用本 Skill 生成发版检查清单。

## 检查范围

1. 版本信息
2. 资源完整性
3. 配置完整性
4. Bundle / 分包
5. 热更新
6. 支付
7. 广告
8. 登录
9. 活动
10. 多语言
11. 埋点
12. 性能
13. 崩溃
14. 平台审核
15. 回滚方案
16. 灰度方案

## 输出格式

```markdown
# 游戏发版检查清单

## 1. 发版信息

| 项目 | 内容 |
|---|---|
| 版本号 | |
| 平台 | Android / iOS / 微信小游戏 / WebGL |
| 发版类型 | 整包 / 热更 / 灰度 |
| 负责人 | |

## 2. P0 必须检查

- [ ] 版本号正确
- [ ] 资源版本号正确
- [ ] 热更新地址正确
- [ ] 支付环境正确
- [ ] 广告位 ID 正确
- [ ] 登录正常
- [ ] 新活动入口正常
- [ ] 奖励发放正常
- [ ] 无明显崩溃

## 3. 资源检查

| 检查项 | 结果 | 备注 |
|---|---|---|
| 图片资源是否缺失 | 待检查 | |
| 图集是否正确 | 待检查 | |

| Bundle 依赖是否完整 | 待检查 | |

## 4. 配置检查

| 检查项 | 结果 | 备注 |

## 5. 热更新检查

- 登录异常
- 奖励不到账

- 热更不可用
- 资源缺失导致主流程不可用
- 活动配置明显错误
```

## 输出要求

- 必须按 P0 / P1 / P2 区分。
- 如果用户提供具体平台，需要重点输出该平台检查项。
- 如果是热更，要重点检查资源版本、CDN、回滚。
- 如果是提审，要重点检查平台审核相关内容。
